E-Mart E-Commerce Platform Setup Guide

This guide will help you set up and run the E-Mart full-stack e-commerce website locally.

Prerequisites

  • Node.js (version 16 or higher) - Download from nodejs.org
  • MongoDB - Install locally or use MongoDB Atlas (cloud). For local: Download from mongodb.com
  • Git (optional, if cloning from repository)

Project Structure

  • client/ - React frontend (Vite)
  • server/ - Express.js backend
  • .env - Environment variables

Setup Steps

  1. Clone or navigate to the project directory: 
    cd c:/Users/iyonicorp/Desktop/emart
  2. Install root dependencies: 
    npm install
  3. Install client dependencies: 
    cd client && npm install && cd ..
  4. Set up environment variables:

    Edit the .env file in the root directory:

    MONGODB_URI=mongodb://localhost:27017/emart
PORT=5000
JWT_SECRET=your_super_secret_jwt_key_change_this_in_production
STRIPE_PUBLIC_KEY=pk_test_your_stripe_public_key
STRIPE_SECRET_KEY=sk_test_your_stripe_secret_key
NODE_ENV=development

    Note: Replace placeholder values with your actual keys. For MongoDB, use your connection string. For Stripe, get test keys from stripe.com.

  5. Start MongoDB:

    If using local MongoDB, start the service:

    mongod

    Or use MongoDB Compass to connect to your database.

Running the Application

Development Mode (Recommended)

Runs both server and client concurrently:

npm run dev
  • Frontend: http://localhost:5173 (Vite dev server)
  • Backend: http://localhost:5000 (Express server)

Production Build

  1. Build the client: 
    npm run build
  2. Start the server: 
    npm start

Individual Commands

  • Run server only: npm run server:dev
  • Run client only: npm run client:dev
  • Preview build: npm run preview

Troubleshooting

  • MongoDB connection error: Ensure MongoDB is running and the URI in .env is correct.
  • Port already in use: Change PORT in .env or kill the process using that port.
  • Dependencies issues: Delete node_modules and package-lock.json, then run npm install again.
  • Stripe payments: Use test keys for development. Real payments require live keys and HTTPS.

Features

  • User authentication (register/login)
  • Product browsing and search
  • Shopping cart
  • Checkout with Stripe integration
  • Order management
  • Admin dashboard

For more details, check the source code in client/src/ and server/ directories.
